# Travel Blog 数据管理指南

本项目采用了**本地文件存储**的方式来管理数据，这意味着你不需要配置复杂的数据库，所有的旅行记录、照片、计划都保存在代码仓库的 `src/data/` 目录下。

这使得部署非常简单（直接推送到 GitHub 即可），同时也方便你通过网页界面来管理内容。

---

## 🚀 快速上手：如何更新内容

### 第一步：开启“编辑模式”

为了安全起见，默认情况下网页是“只读”的。你需要先开启编辑模式才能添加或修改内容。

1.  打开文件：`src/config.js`
2.  将 `IS_EDIT_MODE` 设置为 `true`：
    ```javascript
    export const IS_EDIT_MODE = true;
    ```
3.  保存文件，网页会自动刷新。现在你会发现各个页面上多出了 **Save** 按钮和添加/编辑控件。

### 第二步：在网页上编辑内容

现在你可以像使用普通 App 一样在网页上操作了：

*   **Travel Map (主页)**：
    *   点击地图上的国家 -> "Add Trip" -> 写游记、上传图片。
    *   **保存**：点击左上角的绿色 **Save** 按钮。
*   **Love Gallery (相册)**：
    *   点击 "Add Memory" 上传照片。
    *   可以调整照片的标题、日期和地点。
    *   **保存**：点击上方的绿色 **Save** 按钮。
*   **Future Plans (计划)**：
    *   搜索并添加你想去的目的地。
    *   **保存**：点击右上角的绿色 **Save** 按钮。
*   **Travel Journal (日历)**：
    *   点击日期，写日记，添加心情 Emoji。
    *   **保存**：点击右上角的绿色 **Save** 按钮。

**⚠️ 重要提示**：
点击 **Save** 后，数据会自动写入到你本地电脑的 `src/data/travelData.json` 和 `src/data/galleryData.json` 文件中。你会看到浏览器弹窗提示“Saved successfully”。

### 第三步：部署上线

当你编辑完所有内容后：

1.  **关闭编辑模式**（推荐）：
    *   再次打开 `src/config.js`。
    *   将 `IS_EDIT_MODE` 改回 `false`。这样访客就无法看到编辑按钮了。
2.  **提交代码**：
    *   将所有变动提交到 Git。
    *   `git add .`
    *   `git commit -m "Update travel content"`
    *   `git push`
3.  **自动更新**：
    *   如果你连接了 Vercel，它会自动检测到代码变动并重新部署。几分钟后，你的个人网站就会显示最新的内容了！

## 📂 数据文件说明

如果你更喜欢直接修改代码，或者需要备份数据，可以关注以下文件：

*   `src/data/travelData.json`: 存储了地图游记、未来计划、日历标记等所有核心数据。
*   `src/data/galleryData.json`: 存储了爱心相册的所有照片数据（图片会以 Base64 格式直接存储在里面，或者你可以手动改为引用 `public/images/` 下的路径）。

## 🖼 关于图片

*   **小图片**：通过网页上传的图片会直接转为 Base64 存入 JSON 文件，方便快捷，适合几百 KB 以内的图片。
*   **大图片**：如果你有很高清的大图，建议手动放入 `public/images/` 文件夹，然后手动修改 JSON 文件中的 `src` 字段为 `/images/your-photo.jpg`，这样加载速度更快，也不会让 JSON 文件过大。
